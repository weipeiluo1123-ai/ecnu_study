## 题目核心
- 需求：给定以`@`结尾的表达式字符串，检查其中左右小括号是否匹配（不仅数量相等，还需满足嵌套顺序合法），匹配则输出`YES`，否则输出`NO`
- 考点：栈（stack）的核心用法、括号匹配的经典逻辑、字符串遍历
- 数据范围：表达式长度<255，左括号数量<20

## 70分取巧做法分析
### 核心思路
仅统计左右括号的数量是否相等，未校验括号的嵌套顺序（如`)(`这种顺序错误的情况会被误判为匹配），因此只能通过部分测试用例。

### 70分做法的缺陷
| 测试用例       | 70分做法输出 | 正确输出 | 问题说明                     |
|----------------|--------------|----------|------------------------------|
| `)(@`          | YES          | NO       | 括号顺序错误，数量相等但不匹配 |
| `(2+3))(1-4)@` | YES          | NO       | 右括号提前出现，嵌套顺序非法  |
| `((a+b)@`      | NO           | NO       | 数量不等，结果正确           |

## 核心知识点：栈（stack）
### 1. stack 核心特性
| 特性       | 说明                                  | 注意事项                     |
|------------|---------------------------------------|------------------------------|
| 数据结构   | 后进先出（LIFO）的线性容器            | 仅能访问栈顶元素，无法随机访问 |
| 底层实现   | 基于deque/vector/list（默认deque）    | 不支持迭代器遍历             |
| 空栈操作   | 空栈调用top()/pop()会导致程序崩溃     | 操作前必须用empty()判空      |
| 空间复杂度 | 存储n个元素时空间复杂度为O(n)         | 单类型括号可优化为O(1)       |

### 2. stack 关键方法
| 方法          | 作用                                  | 时间复杂度 | 使用示例                  |
|---------------|---------------------------------------|------------|---------------------------|
| push(x)       | 将元素x压入栈顶                       | O(1)       | `st.push('(');`           |
| pop()         | 弹出栈顶元素（无返回值）              | O(1)       | `if(!st.empty()) st.pop();` |
| top()         | 获取栈顶元素的值                      | O(1)       | `char top_char = st.top();` |
| empty()       | 判断栈是否为空（空返回true）          | O(1)       | `if(st.empty()) isMatch = false;` |
| size()        | 返回栈中元素的个数                    | O(1)       | `int len = st.size();`    |

### 3. 括号匹配的核心逻辑
1. 遍历字符串，遇到左括号`(`时压入栈；
2. 遇到右括号`)`时：
   - 若栈为空（无左括号可匹配），直接判定不匹配；
   - 若栈不为空，弹出栈顶的左括号（完成一次匹配）；
3. 遇到结束符`@`时终止遍历；
4. 遍历结束后：
   - 若栈为空（所有左括号都匹配完成），输出`YES`；
   - 若栈不为空（有未匹配的左括号），输出`NO`。

## 100分正确做法（栈实现）
### 完整AC代码
```cpp
#include <bits/stdc++.h>
#define ios ios::sync_with_stdio(false), cin.tie(0), cout.tie(0);
using namespace std;

// P1739 表达式括号匹配 100分做法（栈实现）
signed main()
{
	ios;
	
	string s;
	cin >> s;
	
	stack<char> st; // 存储左括号的栈
	bool isMatch = true;
	
	for (char c : s) {
		if (c == '@') break; // 遇到结束符停止遍历
		if (c == '(') {
			st.push(c); // 左括号入栈
		} else if (c == ')') {
			if (st.empty()) { // 右括号无左括号匹配
				isMatch = false;
				break;
			}
			st.pop(); // 弹出栈顶左括号，完成匹配
		}
	}
	
	// 遍历结束后，栈为空且无提前匹配失败，才是真正匹配
	if (isMatch && st.empty()) {
		cout << "YES" << '\n';
	} else {
		cout << "NO" << '\n';
	}
	
	return 0;
}
```

### 踩坑点（重点！）（列表+特殊标记）
```markdown
1. ❌ 忽略结束符`@`：遍历整个字符串（包括@之后的字符）→ ✅ 遇到@立即终止遍历
2. ❌ 仅校验数量不校验顺序：70分做法的核心问题 → ✅ 必须用栈维护括号嵌套顺序
3. ❌ 空栈调用pop()/top()：直接导致程序崩溃 → ✅ 操作栈前先调用empty()判空
4. ❌ 遍历结束后未检查栈是否为空：如`((a+b)@`会被误判为YES → ✅ 最终需判断st.empty()
5. ❌ 多余全局变量：保留无意义的全局变量（如int t,n,a[N]）→ ✅ 仅保留必要变量，减少内存占用
```

## 拓展知识点
### 1. 多类型括号匹配拓展（如{}、[]、()）
若题目要求同时匹配多种括号，可通过哈希表映射括号对应关系：
```cpp
#include <bits/stdc++.h>
#define ios ios::sync_with_stdio(false), cin.tie(0), cout.tie(0);
using namespace std;

int main() {
	ios;
	string s;
	cin >> s;
	stack<char> st;
	// 右括号映射到对应的左括号
	unordered_map<char, char> match_map = {{')', '('}, {'}', '{'}, {']', '['}};
	bool isMatch = true;
	
	for (char c : s) {
		if (c == '@') break;
		// 左括号入栈
		if (c == '(' || c == '{' || c == '[') {
			st.push(c);
		}
		// 右括号匹配校验
		else if (match_map.count(c)) {
			if (st.empty() || st.top() != match_map[c]) {
				isMatch = false;
				break;
			}
			st.pop();
		}
	}
	
	if (isMatch && st.empty()) cout << "YES\n";
	else cout << "NO\n";
	return 0;
}
