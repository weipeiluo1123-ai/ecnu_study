## 一、队列核心基础知识
队列（queue）是 C++ STL 中的线性容器，遵循**先进先出（FIFO, First In First Out）** 原则，常用操作及含义如下：

| 方法               | 功能说明                                                                 |
|--------------------|--------------------------------------------------------------------------|
| `q.push(x)`        | 向队列**队尾**插入元素 `x`，队列长度+1                                   |
| `q.pop()`          | 弹出队列**队首**元素（仅删除，无返回值），需先判断队列非空，否则行为未定义 |
| `q.front()`        | 返回队列**队首**元素（仅读取，不删除），需先判断队列非空                 |
| `q.size()`         | 返回队列当前的元素个数，返回值为整数类型                                 |
| `q.empty()`        | 判断队列是否为空：空返回 `true`，非空返回 `false`                        |

### 队列的核心特性
- 只能从队尾添加元素、队首删除/读取元素，无法随机访问中间元素；
- 常用于模拟「排队」场景（如任务排队、消息队列）、广度优先搜索（BFS）等算法。

## 二、本题踩坑点复盘
### 1. 核心错误：`cin.peek()` 的误用
- `cin.peek()` 的作用是「查看输入流的下一个字符（不读取）」，返回的是该字符的 **ASCII 码值**，而非数字本身：
  - 输入数字 `1` 时，`cin.peek()` 得到的是字符 `'1'` 的 ASCII 码（49），而非整数 `1`；
  - 代码中 `cin.peek()==1` 永远不成立，导致「入队指令（1 数字）」完全失效，后续输入处理全部错位，出现「接收不到所有输入」的现象。

### 2. 正确的输入处理逻辑
处理「一行1个/多个数字」的输入时，无需关注「行」的划分：
- C++ 的 `cin` 会自动忽略空格、换行符等空白字符；
- 先读取「操作指令」，再根据指令类型决定是否读取后续参数（如指令1需额外读取入队值，指令2/3/4无需额外参数）。

## 三、关键点总结
1. 队列核心操作需牢记「先进先出」原则，操作前务必用 `q.empty()` 判断队列是否为空，避免非法操作；
2. `cin.peek()` 返回字符的 ASCII 码，切勿直接用其判断数字指令，应先读取整数指令再处理；
3. 多格式输入的核心逻辑：按「指令+参数」的逻辑读取，而非按「行」读取，利用 `cin` 自动跳过空白字符的特性简化处理。

## 最终AC代码
```cpp
#include <bits/stdc++.h>
#define ios ios::sync_with_stdio(false), cin.tie(0), cout.tie(0);
using namespace std;

// B3616 【模板】队列
int main()
{
	ios;
	
	int n;    //操作的次数
	cin >> n;
	
	queue<int> q;
	while(n--){
		int op;
		cin>>op;
		
		if(op==1){    //这里不能用cin.peek() --> ASCII字符 
			int var;
			cin>>var;
			q.push(var);
			continue;
		}
		if(op==2){
			if(q.empty()){
				cout<<"ERR_CANNOT_POP"<<'\n';
			}
			else{
				q.pop();
			}
			continue;
		}
		if(op==3){
			if(q.empty()){
				cout<<"ERR_CANNOT_QUERY"<<'\n';
			}
			else{
				cout<<q.front()<<'\n';
			}
			continue;
		}
		if(op==4){
				cout<<q.size()<<'\n';
			continue;
		}
	}
	
    return 0;
}
```
