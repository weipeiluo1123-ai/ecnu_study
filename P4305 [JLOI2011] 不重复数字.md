### P4305 不重复数字 代码问题总结 & 核心知识点
#### 一、原代码核心问题
1. `unordered_set` 是无序容器，元素存储顺序与输入顺序无关，无法满足题目「保留首次出现顺序」的要求；
2. 仅靠单一容器（set/unordered_set）无法同时实现「去重+保序」。

#### 二、正确解法：容器组合使用
| 容器         | 作用                          | 优势                          |
|--------------|-------------------------------|-------------------------------|
| `unordered_set` | 快速判重（$O(1)$）| 比遍历vector判重效率高        |
| `vector`     | 存储首次出现的数字，保留顺序  | 严格按插入顺序存储            |

#### 三、核心逻辑步骤
1. 遍历输入数字，用 `unordered_set` 判断是否已出现；
2. 未出现的数字：加入vector（保序）+ 插入unordered_set（后续判重）；
3. 遍历vector输出，保证顺序正确且无重复。

#### 四、关键注意点
1. 输出格式：避免末尾多余空格，用 `if(i > 0)` 控制空格输出；
2. 容器选择：
   - 仅去重+排序 → 用set；
   - 仅去重+无序 → 用unordered_set；
   - 去重+保序 → unordered_set + vector；
3. 效率对比：unordered_set判重（$O(1)$）远优于遍历vector判重（$O(n)$）。

### unordered_set 无序性核心说明
你的理解完全正确！`unordered_set` 的核心特点就是「无序」——哪怕你按 `1、2、3、4、5` 的顺序插入，遍历输出的顺序也**完全不确定**，可能是 `1、3、2、5、4`，也可能是 `5、4、3、2、1`，甚至每次运行的输出顺序都可能不一样。

#### 一、为什么 unordered_set 是无序的？
`unordered_set` 底层基于**哈希表**实现：
1. 插入元素时，会根据元素的「哈希值」计算存储位置，而非按插入顺序；
2. 哈希值的计算规则是内部定的，和输入顺序无关，因此元素的存储顺序是“随机”的；
3. 遍历 `unordered_set` 时，是按哈希表的存储位置遍历，而非插入顺序。

#### 二、直观例子验证
```cpp
#include <bits/stdc++.h>
using namespace std;

int main() {
    unordered_set<int> st;
    // 按1、2、3、4、5的顺序插入
    st.insert(1);
    st.insert(2);
    st.insert(3);
    st.insert(4);
    st.insert(5);
    
    // 遍历输出
    for (int x : st) {
        cout << x << " ";
    }
    return 0;
}
```

### 核心疑问解答：unordered_set 为何能降低时间复杂度？
你觉得“本质一样”，是因为从**功能层面**看，都是“判重+插入”，但从**底层实现和操作次数**看，两者有天壤之别——这正是时间复杂度的核心：**相同任务下，需要执行的操作次数不同**。

#### 一、先看一个直观的例子（n=5）
假设输入数字：`2 → 3 → 2 → 5 → 3`，对比两种判重方式的操作次数：

##### 1. 纯vector（遍历判重）
| 输入数字 | 需遍历的vector元素个数 | 总操作次数 | 核心行为                     |
|----------|------------------------|------------|------------------------------|
| 2        | 0（vector为空）| 0          | 插入2 → vector：[2]          |
| 3        | 1（遍历2）| 1          | 插入3 → vector：[2,3]        |
| 2        | 2（遍历2、3）| 2          | 找到重复 → 不插入            |
| 5        | 3（遍历2、3）| 3          | 插入5 → vector：[2,3,5]      |
| 3        | 4（遍历2、3、5）| 4          | 找到重复 → 不插入            |
| **总计** | -                      | **10次**  | -                            |

##### 2. vector + unordered_set（哈希判重）
| 输入数字 | unordered_set判重操作次数 | 总操作次数 | 核心行为                     |
|----------|---------------------------|------------|------------------------------|
| 2        | 1（查哈希表：无2）| 1          | 插入2 → vector：[2]、set：{2}|
| 3        | 1（查哈希表：无3）| 1          | 插入3 → vector：[2,3]、set：{2,3}|
| 2        | 1（查哈希表：有2）| 1          | 不插入                       |
| 5        | 1（查哈希表：无5）| 1          | 插入5 → vector：[2,3,5]、set：{2,3,5}|
| 3        | 1（查哈希表：有3）| 1          | 不插入                       |
| **总计** | -                         | **5次**   | -                            |

#### 二、本质差异：“遍历对比” vs “哈希查找”
##### 1. 纯vector的判重逻辑：「逐个对比」（线性查找）
- 底层是**数组**，判重时必须从第一个元素开始，逐个和目标数字对比，直到找到重复或遍历完；
- 每判重一次，需要的操作次数 = 当前vector的元素个数；
- 总操作次数 = $1+2+3+...+(n-1) = \frac{n(n-1)}{2}$ → 时间复杂度 $O(n^2)$。

##### 2. unordered_set的判重逻辑：「哈希定位」（常数次操作）
- 底层是**哈希表**，每个数字会通过「哈希函数」计算出一个唯一的“位置编号”；
- 判重时，直接根据编号找到对应位置，只需1次操作就能知道“有没有这个数字”；
- 无论哈希表有多少元素，单次判重的操作次数都是**固定常数**（比如1次、2次）→ 时间复杂度 $O(1)$；
- 总操作次数 = $n × 常数$ → 时间复杂度 $O(n)$。

#### 三、用“找东西”的生活例子理解
假设你有1000本书，要找《算法导论》：
- 纯vector方式：从第一本书开始，一本本翻，直到找到（最坏要翻1000本）；
- unordered_set方式：每本书都贴了“编号标签”（哈希值），你知道《算法导论》的标签是500，直接走到500号位置，1秒找到（不管有多少书，都是1步）。

两者的“本质”都是“找书”，但**找的方式不同**，导致花费的时间天差地别——这就是时间复杂度的核心：**不是“做什么”，而是“怎么做”**。

#### 四、时间复杂度的直观对比（n越大，差距越恐怖）
| n（数字个数） | 纯vector操作次数（O(n²)） | unordered_set操作次数（O(n)） | 差距倍数 |
|---------------|---------------------------|-------------------------------|----------|
| 100           | 4950                      | 100                           | 49.5倍   |
| 1000          | 499500                    | 1000                          | 499.5倍  |
| 10000         | 49995000                  | 10000                         | 4999.5倍 |

- 当n=10000时，纯vector需要近5000万次操作，而unordered_set仅需1万次；
- OJ的时间限制通常是1秒（约能执行1亿次操作），n=10000时纯vector已接近超时，n=10万时直接超时，而unordered_set仍游刃有余。

#### 五、总结
1. 功能层面：两者都是“判重”，但**底层实现方式**完全不同（线性遍历 vs 哈希定位）；
2. 操作次数：纯vector的操作次数随n呈“平方级增长”，unordered_set仅“线性增长”；
3. 核心结论：时间复杂度的本质是“操作次数随数据量增长的速率”，而非“是否做了同样的事”——这也是为什么我们需要不同的数据结构：为了用更高效的方式完成相同的任务。


## 最终AC代码
```cpp
#include <bits/stdc++.h>
#define ios ios::sync_with_stdio(false), cin.tie(0), cout.tie(0);
using namespace std;

// P4305 [JLOI2011] 不重复数字
int main()
{
	ios;
	
	int t;
	cin>>t;
	
	while(t--){
		int n;
		cin>>n;
		
		vector<int> myVector;
		unordered_set<int> ust; 
		while(n--){
			int num;
			cin>>num;
			
			if(ust.find(num)==ust.end()){
				myVector.push_back(num);
				ust.insert(num);
			}
			
		}
		for(int x:myVector){
			cout<<x<<" ";
		}
		cout<<'\n';
	}
    return 0;
}
```
