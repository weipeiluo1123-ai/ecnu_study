# 科研工具与工作流

> 面向即将读研的软件工程专硕，辅助论文阅读、信息检索和知识管理。

---

## 一、论文检索（找论文）

### 1. Google Scholar（谷歌学术）
- 地址: https://scholar.google.com
- 用途: 最通用的论文搜索引擎，查作者、查引用、查相关文章
- 技巧: 看引用数判断影响力，被引 > 100 的一般是经典/重要工作

### 2. DBLP（计算机科学论文索引首选）
- 地址: https://dblp.org
- 用途: 计算机科学领域最权威的论文索引数据库
- 特色: 索引准、作者消歧做得好、一键导出 BibTeX
- 使用场景: 嵌入式方向（DAC、ICCAD、EMSOFT、DATE）的论文在 DBLP 上比 Google Scholar 更全更准

### 3. Semantic Scholar
- 地址: https://www.semanticscholar.org
- 用途: AI 增强的学术搜索，比 Google Scholar 多了"影响力评分"和"引用上下文"
- 特色: 直接告诉你这篇论文的核心贡献（TL;DR 摘要）

### 4. Papers with Code
- 地址: https://paperswithcode.com
- 用途: 找论文 + 找开源代码实现的终极站点
- 特色: 每篇论文都关联了 GitHub 代码仓库和 benchmark 排行榜
- 使用场景: 想复现某篇论文时，先来这里搜有没有开源代码

### 5. Connected Papers
- 地址: https://www.connectedpapers.com
- 用途: 输入一篇论文，生成引用关系图谱
- 特色: 直观展示这篇论文的前驱工作和后续发展，适合了解研究脉络

### 6. ArXiv
- 地址: https://arxiv.org
- 用途: 预印本平台，最新论文第一时间发布在这里
- 技巧: 按分类订阅（cs.AI, cs.CL, cs.SE, cs.LG 等），每天接收新论文推送

### 7. CCF DDList（截稿日期追踪）
- 地址: https://ccfddl.com
- 用途: 查看所有 CCF 推荐会议的截稿日期，按时间倒排序
- 特色: 嵌入式方向（EMSOFT、DATE、ASP-DAC、CODES+ISSS）的 deadline 一目了然，支持添加日历提醒
- 使用场景: 规划论文投稿时间线时必备

## 二、如何判断是不是顶会

计算机领域主要看会议，不是看期刊。常见顶会：

| 方向 | 顶会 |
|------|------|
| 机器学习 | **NeurIPS**, ICML, ICLR |
| 计算机视觉 | **CVPR**, ICCV, ECCV |
| 自然语言处理 | **ACL**, EMNLP, NAACL |
| 人工智能 | **AAAI**, IJCAI |
| 系统/网络 | **SOSP**, OSDI, SIGCOMM |
| 软件工程 | **ICSE**, FSE, ASE |
| 数据库 | **SIGMOD**, VLDB, ICDE |
| 安全 | **S&P**, CCS, USENIX Security |

怎么快速判断：**看论文发表在哪里**。Google Scholar / Semantic Scholar 会在论文条目下显示会议名，看到上面的缩写基本就是顶会。

## 三、论文阅读方法

### 推荐流程：三遍阅读法

**第一遍（5 分钟）**：
- 读标题、摘要、结论
- 扫一眼图表
- 回答：这篇论文讲什么？和我相关吗？

**第二遍（30 分钟）**：
- 读 Introduction + 图表 + 结论
- 跳过数学推导和实验细节
- 回答：方法的核心思路是什么？实验结果支持结论吗？

**第三遍（1 小时+）**：
- 读完整论文，包括公式和实验设置
- 回答：我能不能复现？有什么可以改进的？

### 读论文的目标
- **不要试图理解每个单词**，先抓住核心 idea
- **看不懂公式先跳过**，不影响理解整体思路
- **带着问题读**：这篇解决了什么问题？怎么解决的？有什么局限？

### 精读工具推荐

- **ReadPaper** (https://readpaper.com) — 中文界面、PDF 阅读 + 翻译 + 笔记一体、社区有论文讨论，对有公式推导的论文（形式化方法、硬件设计等）支持较好
- **Zotero PDF Translate 插件** — 在 Zotero 里直接划词翻译，不用切到浏览器，适合集中阅读

## 四、工具链

### 论文管理

**Zotero（最推荐）**
- 官网: https://www.zotero.org
- 功能:
  - 一键抓取论文元数据（标题、作者、期刊、摘要）
  - 分类管理（文件夹 + 标签）
  - 插件生态丰富：自动翻译、PDF 标注、笔记
  - 支持插件市场安装，推荐插件: Zotero PDF Translate, Better BibTeX for Zotero
- 使用方式: 安装浏览器插件，在论文页面一键保存

**Zotero + Obsidian 组合（论文管理 + 知识网络）**

这是目前很多 SE lab 推荐的科研工作流：

```
Zotero 负责：论文元数据 + PDF 管理 + 标注
Obsidian 负责：阅读笔记 + 知识链接 + 双向图谱

通过 Zotero Integration 插件打通：
  在 Obsidian 里一键导入 Zotero 条目
  笔记自动关联回 Zotero 里的 PDF
  Obsidian 的图谱视图能直观看到论文之间的引用关系
```

好处是读论文时随手记下的 idea、related work 对比、实验想法可以互相链接，形成知识网络。写文献综述时可以直接从 Obsidian 里整理出来。

**备选：Notion 数据库**
- 适合做「论文阅读清单」：加状态字段（待读 / 精读中 / 已整理）
- 数据库视图可以按会议、年份、标签筛选
- 但 PDF 管理不如 Zotero，建议 Zotero 管文件 + Notion 管进度

### 论文阅读与标注

**方案 A: Zotero 内置 PDF 阅读器**
- 高亮、标注、笔记都在论文旁边
- 笔记可与论文条目关联

**方案 B: 沉浸式翻译插件配合网页阅读**
- 浏览器插件: 沉浸式翻译（Immersive Translation）
- 在 Semantic Scholar / ArXiv 页面直接显示双语对照
- 适合快速浏览

**方案 C: ReadPaper（精读首选）**
- 对嵌入式方向带公式推导的论文支持更好（形式化方法、硬件设计等）
- 中文翻译、PDF 标注、笔记一体
- 有社区讨论功能

### 科研论文写作

**Overleaf（LaTeX 在线协作编辑器）**
- 地址: https://www.overleaf.com
- 用途: 嵌入式方向论文几乎都用 LaTeX 排版，Overleaf 是标配
- 特色: 在线编辑 + 实时编译 + 多人协作，会议模板从官网下载后直接导入
- 技巧: 用 Better BibTeX for Zotero 导出 .bib 文件，Overleaf 里一键引用

### 架构图 / 流程图

**draw.io (diagrams.net)**
- 地址: https://app.diagrams.net
- 用途: 画论文架构图、流程图、系统框图
- 特色: 免费、支持导出 SVG/PDF（矢量图，论文插图质量标准）
- 技巧: 嵌入 Obsidian 有原生插件，可以在笔记和论文图之间快速切换

### 知识管理

**写笔记比收藏重要得多**

推荐用你熟悉的工具，不要为了记笔记学一个新工具：
- 用 Blog 平台写论文阅读笔记（顺便填充你的项目）
- 用 Markdown 文件管理，Git 版本跟踪
- 每篇论文至少写 3 句话：解决了什么问题 / 方法核心 / 你觉得有意思的点

## 五、每日工作流建议（15 分钟）

```
第一步: 打开 Semantic Scholar 或 ArXiv 订阅
         → 扫一眼当天推送的新论文标题和摘要
         → 选 1 篇感兴趣的（5 分钟）

第二步: 读摘要 + 结论 + 扫图表
         → 用"三遍阅读法"的第一遍
         → 判断是否值得精读（5 分钟）

第三步: 如果相关，保存到 Zotero
         → 写一句话记录："这篇做了什么，和我有什么关系"（5 分钟）
```

坚持 3 个月，你会发现自己对本领域的认知远超周围同学。

## 七、推荐工作流组合（嵌入式方向研究生）

```
检索:   Google Scholar + DBLP + CCF DDList（看 deadline）
管理:   Zotero + Obsidian（Zotero Integration 插件打通）
阅读:   Zotero PDF Translate（划词翻译）+ ReadPaper（精读）
笔记:   Obsidian（双向链接，形成知识网络）
写作:   Overleaf（LaTeX 在线协作）
画图:   draw.io（架构图、流程图）
```
