## 新学知识 std::map<key_type, value_type> myMap;
### map 核心特性与方法总结（细节版）
#### 一、map 核心特性
map 是 C++ STL 中的**关联式容器**，底层基于红黑树实现，核心特性如下：
1. **键值对（key-value）存储**：每个元素由「唯一的 key」和对应的「value」组成，key 与 value 类型可不同（如 `map<int, string>`）。
2. **自动排序**：会按照 key 的**升序**自动排序（默认用 `<` 比较），无法手动打乱顺序；若需降序，可指定比较规则（如 `map<int, string, greater<int>>`）。
3. **key 唯一性**：不允许重复的 key，若插入已存在的 key，新 value 会覆盖旧 value，且插入操作返回的 pair 中 `second` 为 `false`。
4. **非连续存储**：底层红黑树结构决定其元素并非连续内存，无法用下标随机访问（虽支持 `[]` 操作，但本质是查找/插入，非数组式访问）。
5. **查找效率高**：基于 key 查找的时间复杂度为 $O(logn)$，远优于线性容器（如 vector）。

#### 二、map 核心方法（细节重点）
##### 1. 插入操作
| 方法                | 细节说明                                                                 |
|---------------------|--------------------------------------------------------------------------|
| `map[key] = value`  | 最便捷的插入方式：若 key 不存在则新增键值对；若已存在则覆盖 value。       |
| `insert({{k1,v1}, {k2,v2}})` | 批量插入，不会覆盖已有 key 的 value；插入重复 key 时，该次插入无效（返回值可判断）。 |
| `emplace(k, v)`     | 原地构造键值对，效率比 insert 更高（减少拷贝），同样不覆盖已有 key。      |

##### 2. 查找操作
| 方法                | 细节说明                                                                 |
|---------------------|--------------------------------------------------------------------------|
| `find(key)`         | 返回指向该 key 对应键值对的迭代器；若 key 不存在，返回 `map.end()`（尾后迭代器，不可解引用）。 |
| `count(key)`        | 返回 key 存在的数量（仅 0 或 1，因为 key 唯一），常用于快速判断 key 是否存在。 |
| `lower_bound(key)`  | 返回第一个「key ≥ 指定值」的迭代器；`upper_bound(key)` 返回第一个「key > 指定值」的迭代器。 |

##### 3. 删除操作（重点细节）
| 方法                | 细节说明                                                                 |
|---------------------|--------------------------------------------------------------------------|
| `erase(key)`        | **核心：根据 key 删除整个 key-value 键值对**，返回删除的元素个数（map 中仅 0 或 1）。 |
| `erase(iterator)`   | 删除迭代器指向的键值对，需确保迭代器有效（非 `end()`），无返回值。        |
| `erase(it1, it2)`   | 删除 [it1, it2) 区间内的所有键值对，左闭右开，无返回值。                 |
| `clear()`           | 清空所有键值对，map 变为空，size 变为 0。                                |

> 注意：erase 仅删除容器内的键值对，不会释放 key/value 本身的内存（若 value 是指针，需手动释放）。

##### 4. 访问与遍历
| 方法                | 细节说明                                                                 |
|---------------------|--------------------------------------------------------------------------|
| `map[key]`          | 访问 key 对应的 value：若 key 不存在，会自动插入该 key，且 value 为默认值（如 int 为 0，string 为空）。 |
| `at(key)`           | 访问 key 对应的 value：若 key 不存在，直接抛出 `out_of_range` 异常（比 `[]` 更安全）。 |
| 迭代器遍历          | `for (auto it = map.begin(); it != map.end(); ++it)`，`it->first` 是 key，`it->second` 是 value。 |
| 范围 for 遍历       | `for (auto& [k, v] : map)`（C++17 及以上），直接访问 key 和 value，建议加 `&` 避免拷贝。 |

##### 5. 其他常用方法
| 方法                | 细节说明                                                                 |
|---------------------|--------------------------------------------------------------------------|
| `size()`            | 返回键值对的个数，时间复杂度 $O(1)$。                                    |
| `empty()`           | 判断是否为空，空返回 `true`，非空返回 `false`，时间复杂度 $O(1)$。        |
| `swap(map2)`        | 交换两个 map 的内容（包括所有键值对），时间复杂度 $O(1)$（仅交换底层指针）。 |

#### 三、易踩坑细节补充
1. `erase` 迭代器时，若遍历中删除元素，需注意：`map.erase(it++)` 是安全写法（先返回迭代器副本，再自增，避免迭代器失效）；直接 `erase(it)` 会导致后续 `++it` 访问非法内存。
2. `map[]` 是「查找+插入」操作，若仅想判断 key 是否存在，用 `count()` 或 `find()` 更高效（避免不必要的插入）。
3. map 的迭代器不会因插入/删除（除被删除的迭代器外）而失效，这是红黑树结构的特性。

## 最终AC代码
```cpp
#include <bits/stdc++.h>
#define ios ios::sync_with_stdio(false), cin.tie(0), cout.tie(0);
using namespace std;

const int Q = 1e5+10;

// B3616 【模板】队列
int main()
{
	ios;
	
	int n;
	cin>>n;
	
	map<string, int> mp;
	while(n--){
		int op;
		cin>>op;
		
		if(op==1){
			string name;
			int score;
			cin>>name>>score;
			mp[name] = score;    //插入/修改 
			cout<<"OK"<<'\n';
			continue;
		}
		if(op==2){
			string name;
			cin>>name;
			bool exist = mp.count(name);
			if(exist){
				cout<<mp[name]<<'\n';
			}else{
				cout<<"Not found"<<'\n';
			}
			continue;
		}
		if(op==3){
			string name;
			cin>>name;
			bool exist = mp.count(name);
			if(exist){
				mp.erase(name);
				cout<<"Deleted successfully"<<'\n';
			}else{
				cout<<"Not found"<<'\n';
			}
			continue;
		}
		if(op==4){
			cout<<mp.size()<<'\n';
			continue;
		}
	}
	
    return 0;
}
```
