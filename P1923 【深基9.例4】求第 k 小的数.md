# 【算法题】P1923 【深基9.例4】求第 k 小的数
## 一、题目核心逻辑
输入 n 个整数，找出其中第 k 小的数（默认 k 从 0 开始计数，若题目要求从 1 开始需调整为`k-1`）。

## 二、初始代码问题分析

### 1. 初始错误代码

```cpp

#include <bits/stdc++.h>
using namespace std;

int main(){
	int n, k; 
	cin >> n >> k;
	
	multiset<int> mt_set;
	while(n--){
		int num;
		cin >> num;
		mt_set.insert(num);
	}
	auto it = mt_set.begin();
	cout << mt_set[k] << '\n'; // 核心错误
	
	return 0;
}
```

### 2. 错误点汇总

|错误类型|具体问题|
|---|---|
|**语法错误**|`multiset` 不支持`[]`下标访问（仅随机访问迭代器容器支持），需通过迭代器移动 + 解引用获取值|
|**效率问题**|`multiset`插入 / 遍历常数大，大数据量下会超时（TLE）|
|**编译环境问题**|易出现`ld returned 1 (Permission denied)`，因 exe 文件被占用导致链接失败|
## 三、关键知识点解析

### 1. C++ 迭代器类型与操作

C++ 迭代器分为不同类型，决定了支持的操作：

|迭代器类型|支持容器|支持操作|不支持操作|
|---|---|---|---|
|随机访问迭代器|`vector`/`array`/ 原生数组|`+k`/`-k`/`[]`/`advance`|-|
|双向迭代器|`multiset`/`set`/`list`|`++`/`--`/`advance`|`+k`/`-k`/`[]`|
- `advance(it, k)`：通用迭代器移动函数，属于`<iterator>`头文件（万能头`bits/stdc++.h`已包含），自动适配迭代器类型：

    - 随机访问迭代器：等价于`it += k`（时间复杂度 O (1)）

    - 双向迭代器：等价于循环 k 次`++it`（时间复杂度 O (k)）

### 2. `sort`算法使用说明

- 所属头文件：`<algorithm>`（万能头`bits/stdc++.h`已包含）

- 函数原型：`sort(RandomIt first, RandomIt last)`

- 适用范围：仅支持**随机访问迭代器**（`vector`/`array`/ 原生数组）

- 不适用容器的处理方案：

    - `list`：使用自身成员函数`ls.sort()`

    - `multiset`/`set`：本身自动排序，无需手动调用`sort`

    - 通用方案：将容器元素拷贝到`vector`排序后，再拷贝回原容器

## 四、正确解法（按效率排序）

### 1. 基础修复版（解决语法错误，仍可能超时）

```cpp

#include <bits/stdc++.h>
using namespace std;

int main(){
    int n, k; 
    cin >> n >> k;
    
    multiset<int> mt_set;
    while(n--){
        int num;
        cin >> num;
        mt_set.insert(num);
    }
    
    auto it = mt_set.begin();
    advance(it, k); // 替换[]访问，用advance移动迭代器
    cout << *it << '\n'; // 解引用迭代器获取值
    
    return 0;
}
```

### 2. 高效推荐版（`vector + sort`）

兼顾简洁与效率，解决超时问题：

```cpp

#include <bits/stdc++.h>
using namespace std;

int main(){
    // 加速cin/cout，处理大数据输入必备
    ios::sync_with_stdio(false);
    cin.tie(0);
    
    int n, k; 
    cin >> n >> k;
    
    vector<int> nums(n);
    for(int i = 0; i < n; i++){
        cin >> nums[i];
    }
    
    sort(nums.begin(), nums.end()); // STL优化快排，效率远高于multiset
    cout << nums[k] << '\n'; // 随机访问O(1)
    
    return 0;
}
```

### 3. 最优版（快速选择 QuickSelect，平均 O (n) 复杂度）

适合超大数据量场景（如 n≥1e6）：

```cpp

#include <bits/stdc++.h>
using namespace std;

// 快速选择核心：找出nums[left..right]中第k小的数
int quickSelect(vector<int>& nums, int left, int right, int k) {
    if (left == right) return nums[left];
    
    int pivot = nums[right]; // 选最右元素作为基准
    int i = left - 1; // 小于基准的区域指针
    
    // 分区：将小于等于pivot的元素移到左边
    for (int j = left; j < right; j++) {
        if (nums[j] <= pivot) {
            swap(nums[++i], nums[j]);
        }
    }
    swap(nums[++i], nums[right]); // 将基准值放到正确位置
    
    // 根据基准位置递归查找
    if (i == k) return nums[i];
    else if (i < k) return quickSelect(nums, i+1, right, k);
    else return quickSelect(nums, left, i-1, k);
}

int main(){
    ios::sync_with_stdio(false);
    cin.tie(0);
    
    int n, k; 
    cin >> n >> k;
    
    vector<int> nums(n);
    for(int i = 0; i < n; i++){
        cin >> nums[i];
    }
    
    cout << quickSelect(nums, 0, n-1, k) << '\n';
    
    return 0;
}
```

## 五、常见问题解决方案

### 1. 编译报错：`ld returned 1 (Permission denied)`

- **原因**：exe 文件被占用（程序未正常退出、Dev-C++ 自动运行残留）

- **解决步骤**：

    1. 按`Ctrl+Shift+Esc`打开任务管理器，结束对应 exe 进程（如`题单2.exe`）

    2. 重启 Dev-C++，或点击「运行」→「清理工程」删除旧文件

    3. 取消 Dev-C++ 的「编译后自动运行」选项（工具→编译选项）

### 2. 运行超时（TLE）

- **原因**：`multiset`插入 / 遍历的常数开销大，时间复杂度为 O (nlogn + k)

- **解决方法**：

    1. 替换为`vector + sort`方案（时间复杂度 O (nlogn)，常数开销小）

    2. 超大数据量使用快速选择算法（平均 O (n) 复杂度）

    3. 必加优化：`ios::sync_with_stdio(false); cin.tie(0);`加速输入输出

## 六、核心总结

1. **迭代器操作**：`multiset`仅支持双向迭代器，需用`advance(it, k)`移动，不可用`[]`或`+k`；

2. **效率选择**：刷题优先使用`vector + sort`，兼顾简洁与效率，大数据量可选快速选择；

3. **环境问题**：exe 文件占用导致的编译错误，需结束进程或清理工程。
